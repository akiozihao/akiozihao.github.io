使用 gdb debug xv6

  1. make CPUS=1 qemu-gdb 启动qemu，打开 gdb-server，可以看到这里server的端口号是25501
  2. 另起terminal /opt/homebrew/bin/riscv64-elf-gdb 打开 gdb-client
  3. target remote 127.0.0.1:25501 与 server 建立连接
  4. b _entry 设置入口断点
  5. c 跳转到入口处，可以发现代码没有停在0x8000000,而是停在0x8000000a，kernel/kernel.sam里可以看到 8000000a: f14025f3 csrr a1,mhartid 读取了系统寄存器 mhartid 并将结果加载到了 a1 寄存器中。0x8000000是被qemu认可的地址，使用qemu的时候第一个指令地址必须是这个特定地址。kernel/kernel.ld里可以看到是从这个地址开始读
  6. si 执行下一条指令
  7. b main 在main.c 打点
  8. layout split 分割视图
  9. n next
  10. p print
  11. s step into
  12. del 3 删除断点3
